# MkDocs documentation series

A structured, multi-part documentation series that explains how to build, manage, and deploy documentation using MkDocs — designed specifically for real-world documentation workflows, not just basic setup.

This series is written in clear, practical language and focuses on how technical writers and documentation teams actually work in Docs-as-Code environments.

---

## Who this series is for

This series is written for:

- Technical writers working in or transitioning to Docs-as-Code environments  
- Documentation teams building scalable and maintainable documentation portals  
- Engineers or product teams responsible for owning and maintaining product documentation  
- Anyone using MkDocs without a clear understanding of structure, workflows, and best practices  

No prior experience with MkDocs is assumed, but a basic understanding of Markdown is helpful.

---

## What you will learn

By the end of this series, you will understand:

- How MkDocs works as a documentation framework  
- How to structure documentation projects for clarity and scalability  
- How to configure `mkdocs.yml` for real-world use cases  
- How to design navigation and improve documentation usability  
- How to apply theming and customization effectively  
- How to implement plugins and extend functionality  
- How to set up local development workflows for documentation  
- How to deploy documentation using GitHub Pages  
- How to automate documentation workflows using CI/CD pipelines
